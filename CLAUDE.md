## 4. Tauri + Next 16 / React 19 traps

- **`react-hooks/set-state-in-effect` is a hard lint error** (Next 16). Never `setState(...)` inside `useEffect` synchronously. Patterns that work:
  - lazy `useState(() => { if (typeof window === 'undefined') return X; ... })` with SSR guard
  - `useSyncExternalStore` for storage-backed reactive values (we use it for `showSplash`)
  - `queueMicrotask(() => { void fetch...(); })` defers past render commit
- **Dark mode via Tailwind 4**: requires `@custom-variant dark (&:where(.dark, .dark *));` in `globals.css`. Without it, `dark:` follows `prefers-color-scheme` not `.dark` class
- **Theme class pre-hydrate**: use `next/script` with `strategy="beforeInteractive"` in `layout.tsx`. Plain `<script>` tags in React tree are silently dropped in Next 16
- **Sun/Moon icon swap**: CSS-only via `dark:hidden` / `hidden dark:block` to avoid hydration mismatch from structurally different JSX
- **Splash via `createPortal(..., document.body)`** with z-9999 â€” covers sidebar regardless of render tree position
- **Inline `<img>` for SVG/PNG**: add `{/* eslint-disable-next-line @next/next/no-img-element */}` to silence

## 5. Tauri sidecar (PyInstaller backend)

- **`bundle.externalBin: ["binaries/sovlens-backend"]`** â€” Tauri auto-appends target triple (e.g. `-aarch64-apple-darwin`)
- **Build scripts**: `frontend/src-tauri/binaries/build.sh` (mac/linux), `build.bat` (win). Each runs PyInstaller via `backend/sovlens-backend.spec` and renames the output binary
- **`frontend/src-tauri/binaries/sovlens-backend-*`** are gitignored. CI builds them per-platform. Locally you must build before `npm run tauri build`
- **Lib.rs spawn sidecar**: only in release (`#[cfg(not(debug_assertions))]`). Dev mode runs `python main.py` separately
- **Sidecar binary size guard**: `lib.rs` skips spawn if binary < 4 KB (prevents panic on empty placeholder)

## 8. Auto-updater (Tauri 2)

- **`bundle.createUpdaterArtifacts: true` REQUIRED** in `tauri.conf.json`. Without it, `tauri build` never emits `.sig` files and `tauri-action` skips `latest.json` upload with "Signature not found for the updater JSON"
- Env vars: `TAURI_SIGNING_PRIVATE_KEY` + optional `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`. These are read by `tauri-cli`, NOT by `tauri-action`
- `pubkey` in `tauri.conf.json` is base64-wrapped full minisign pub file (multi-line decoded)
- Private key generated via `npx tauri signer generate --ci -w ~/.tauri/sovlens-updater.key`
- Local key backup: `secrets/Auto Update Key For SovLens` (gitignored via `secrets/`)
- GitHub Actions: **Repository secret** named `TAURI_SIGNING_PRIVATE_KEY` (Environment secrets are invisible to our workflow)
- Endpoint: `https://github.com/asifwanders/SovLens/releases/latest/download/latest.json`

## 14. Common mistakes I keep making

1. **`hover:bg-black/5` is invisible in light mode.** Use `hover:bg-black/15 hover:text-accent` for buttons
2. **`bundle.targets: "all"`** triggers DMG on mac which fails on CI. Use explicit list
3. **Setting `setState` in `useEffect`** triggers Next 16 lint error
4. **PyInstaller binary path mismatch** â€” sidecar must be at `binaries/sovlens-backend-{target-triple}` exactly
5. **Forgetting to delete + re-tag** â€” workflow only triggers on tag push, not main push
6. **Pasting key contents with surrounding whitespace** into GitHub Secrets
7. **`bundle.createUpdaterArtifacts`** â€” easiest single config to forget
8. **`uvicorn.run("main:app", reload=True)` in PyInstaller frozen exe** â€” reloader re-execs the bootloader â†’ backend never binds. Use `uvicorn.run(app, reload=False)` with the app object, not import string
9. **Dropping `_rx` from `sidecar.spawn()`** â€” discards stdout/stderr forever. ALWAYS pipe to `backend.log` via `tauri::async_runtime::spawn` reading `CommandEvent::Stdout/Stderr/Error/Terminated`
10. **`localhost` on Windows resolves IPv6 `::1` first** â€” uvicorn binds IPv4 only â†’ 1-2s delay or refused. Use `127.0.0.1` in all frontend fetch URLs
11. **Tauri file picker `filters` with too many extensions in one group** â€” Windows `IFileDialog` may reject. Split into multiple filter groups + add `All Files: *` fallback
12. **`app.path().resolve("sovlens-backend", Resource)` returns bare name without `.exe` on Win** â€” bundled file is `sovlens-backend.exe` so metadata() fails â†’ 4KB guard skips spawn. Never pre-flight by literal name; let `app.shell().sidecar(name)` do OS-aware resolution and log any error
13. **PyInstaller `excludes=["unittest"]` crashes torch on import** â€” `torch.utils._config_module` does `import unittest` at module load. Never exclude `unittest` from the spec. Defensively force-include `unittest`, `unittest.mock`, `pkg_resources`, `sqlite3`, `importlib.metadata`
14. **Hand-listed `hiddenimports` for torch is fragile** â€” use `collect_all('torch')` (and torchvision/transformers/ultralytics/easyocr/whisper/lancedb/scenedetect) so lazy submodules + DLLs come along
15. **PyInstaller `upx=True` on Win** â€” Defender heuristic flag + corrupts torch DLLs. Always `upx=False` for Win bundles with torch
16. **Subprocess spawned from PyInstaller `console=False` GUI app pops `conhost.exe` window on each child** â€” set `creationflags=0x08000000` (CREATE_NO_WINDOW) in `subprocess.run` on Windows
17. **Tauri NSIS defaults to `perMachine`** requiring admin â†’ SmartScreen + UAC double-prompt. Set `bundle.windows.nsis.installMode: "currentUser"`
18. **Tauri NSIS default template kills main exe but NOT sidecar** â€” upgrade installer fails with "Error opening file for writing: ...sovlens-backend.exe". Add `installerHooks: "./windows/hooks.nsh"` with `NSIS_HOOK_PREINSTALL` + `NSIS_HOOK_PREUNINSTALL` macros running `taskkill /F /IM sovlens-backend.exe /T`
19. **Tauri 2 plugin commands need explicit capability ACL** â€” `@tauri-apps/plugin-updater`'s `check()` IPC blocked by default. Add `updater:default` (covers allow-check + allow-download-and-install) to `capabilities/default.json` permissions. Same pattern for every JS plugin call: missing perm = "Command plugin:X|Y not allowed by ACL". Server-side `app.updater()` from a Rust `#[tauri::command]` bypasses ACL â€” ACL only gates webviewâ†’Rust IPC
20. **Settings UI `warming: string | null` blocks parallel model downloads** â€” switching to Extreme needs whisper+ocr+yolo concurrently. Use `Set<string>` + `Set<string>` ref; iterate Set to drop keys once `/models/status` reports `downloaded: true`. Always-poll `/models/status` while page mounted (cheap, no model I/O) â€” gating poll on `warming` misses background downloads kicked off elsewhere
21. **CUDA torch on Win silently downgrades to CPU wheel during `pip install -r requirements.txt`** â€” pip's resolver replaces the +cu121 wheel with the plain PyPI CPU wheel when any transitive dep asks for `torch` without the local-version tag. Install order alone is not enough. ALWAYS re-pin AFTER main requirements: `pip install --force-reinstall --no-deps torch --index-url https://download.pytorch.org/whl/cu121` + hard-fail CI on `assert torch.version.cuda`. Also log `cuda.is_available`, `version.cuda`, `device_name` at backend startup so this is visible in `backend.log` post-install

## 16. Sub-agent strategy that works

- **HARD RULE**: ALWAYS spawn sub-agents for ANY of:
  - Documentation lookups (Tauri, PyInstaller, third-party crates, HF, etc.)
  - Cross-file scans / grep across >1 file
  - Multi-file edits where agents can work in parallel
  - Audits / reviews
  - Researching error messages or library API misuse
  - Reading >2 unfamiliar files to understand a bug
- **User has called this out twice â€” keep main context clean by default**
- Do NOT inline read+grep+web research in main thread when a single-shot agent could return a 200-word summary instead of 50KB of raw file content
- Spawn parallel agents when slices are disjoint; serialize when same file
- Phase A: 3-5 agents in parallel on disjoint files
- Phase B: serialize agents that touch same file
- Phase C: 1 Opus reviewer (different model than implementers)
- Phase D: 1 Sonnet fixer for review findings
- Phase E: 1 Opus final sweep
- Always Task-track via TaskCreate/TaskUpdate to survive /compact

### Anti-patterns I keep falling into
- Reading 3+ files directly to "verify the bug" instead of asking an agent to do the diagnosis
- Doing edits directly when a sub-agent could batch-edit + report back
- Using WebFetch in main thread for docs research â€” always delegate to an agent
